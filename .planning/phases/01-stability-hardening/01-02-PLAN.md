Add error handling pattern to all ViewModels:

1. HomeViewModel.kt:
   - Add `_errorMessage = MutableLiveData<String?>()` and `val errorMessage: LiveData<String?>`
   - Add `fun clearError() { _errorMessage.value = null }`
   - In the Firestore catch block (line 72-74), set error message instead of silently returning null:
     ```kotlin
     _errorMessage.value = "Couldn't load dashboard — check your connection"
     ```
   - Add auto-retry: wrap the Firestore call in a try block that retries once after 2s delay:
     ```kotlin
     try {
         semesters = FirestoreSemesterRepository.getSemesters(deviceUid, profileId)
     } catch (e1: Exception) {
         delay(2000)
         try {
             semesters = FirestoreSemesterRepository.getSemesters(deviceUid, profileId)
         } catch (e2: Exception) {
             _errorMessage.value = "Couldn't load dashboard — check your connection"
         }
     }
     ```
   - Add `import kotlinx.coroutines.delay`
   - Keep existing SWR cache pattern (cachedDashboard) untouched

2. ProfileViewModel.kt:
   - Add `_errorMessage = MutableLiveData<String?>()` and `val errorMessage: LiveData<String?>`
   - Add `fun clearError() { _errorMessage.value = null }`
   - In the Firestore catch block (line 54-57), set error message:
     ```kotlin
     _errorMessage.value = "Couldn't load profile — check your connection"
     ```
   - Add auto-retry with 2s delay (same pattern as HomeViewModel)

3. SemViewModel.kt:
   - Already has `_errorMessage` StateFlow. No structural changes needed.
   - **Change:** Replace the Snackbar-style error message with the inline card pattern. The error already uses `_errorMessage.value`, which the UI will now display via the error card instead of a Snackbar.
   - **Add:** Auto-retry in `loadSemester()` catch block (lines 109-111):
     ```kotlin

         delay(2000)
         try {
             // retry the Firestore call
         } catch (e2: Exception) {

         }
     }
     ```
   - Ensure `loadSemester()` doesn't silently swallow errors (it currently just sets `_isLoading = false`)
